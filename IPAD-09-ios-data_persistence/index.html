import UIKit
class ViewController: UIViewController {
  
  override func viewDidLoad() {
    super.viewDidLoad()
    writeData()
    readData()
  }
  
  func writeData() {
    // data to be written
    let sampleData = ["a", "b", "c"]
    // create url to the file
    let url = FileManager.default.urls(for: .documentDirectory, 
      in: .userDomainMask)[0].appendingPathComponent("sample.plist")
    // create and configure the encoder
    let encoder = PropertyListEncoder()
    encoder.outputFormat = .xml
    do {
      // encode the data
      let data = try encoder.encode(sampleData)
      // write the data
      try data.write(to: url)
    } catch {
      print(error)
    }
  }
  
  func readData() {
    // create url to the file
    let url = FileManager.default.urls(for: .documentDirectory, 
      in: .userDomainMask)[0].appendingPathComponent("sample.plist")
    // create the decoder
    let decoder = PropertyListDecoder()
    do {
      // read the data
      let xml = try Data(contentsOf: url)
      // decode the data
      let sampleData = try decoder.decode([String].self, from: xml)
      // print the data for debugging purposes
      print(sampleData)
    } catch {
      print(error)
    }
  }
}

Error handling is the process of responding to and recovering from error conditions

  • some operations may fail due to foreseen or unforeseen circumstances
  • when an operation fails, try to understand what caused the failure and respond accordingly

For example, consider the task of reading and processing data from a file on disk; this task can fail if the file:

  • does not exist at the specified path
  • does not have read permissions

Distinguishing between these error conditions allows a program to respond accordingly and communicate appropriately to the user

Representing Errors

Errors are represented by values of types that conform to the Error protocol

Throwing Errors

Throwing an error lets you indicate that something unexpected happened and the normal flow of execution cannot continue. You use a throw statement to throw an error.

Handling Errors

When an error is thrown, some surrounding piece of code must be responsible for detecting and handling the error.

  1. You can use a do-catch construct - the do block contains the statement which has a try while the catch block handles the error; note that all code in the do block following the code which had an error will be bypassed
  2. You can use try? - the error is discarded, but an error is indicated by the presence of a nil return value; note that the return type automatically becomes an optional type regardless of the declaration of the function being called
  3. You can use try! - this assumes that an error will never occur, but if it does, the application will crash

iOS file system

iOS devices have an hierarchical file system, just like Windows and macOS machines.

For security purposes, an iOS app has a limited number of places where it can write its data. When an app is installed on a device (either from iTunes or the App Store), the system creates a number of containers for the app. These containers represent the universe for that app and contain everything the app can access directly. One such container is the Documents directory, which is used to store user-generated content. The contents of this directory can be made available to the user through file sharing; therefore, this directory should only contain files that you may wish to expose to the user.

On an iOS device, the path to the Documents directory looks similar to the path shown below.

/var/mobile/Containers/Data/Application/01C10AB7-78FF-4078-85E4-780FDF8D700E/Documents/

On an iOS simulator, the filesystem of the iOS device maps onto the host macOS file system and looks similar to the path shown below. This allows us to inspect the files in our app.

FileManager

A file manager object lets you examine the contents of the file system and make changes to it. The FileManager class provides convenient access to a shared file manager object that is suitable for most types of file-related manipulations. A file manager object is typically your primary mode of interaction with the file system. You use it to locate, create, copy, and move files and directories. You also use it to get information about a file or directory or change some of its attributes.

Property lists (plist) are stored as XML or binary and can be used to store:

  • primitive values - strings, numbers, binary data, dates, and Boolean values
  • containers of values - arrays and dictionaries; containers can contain other containers as well as primitive types. For example, you could have an array of dictionaries, or a dictionary of arrays, or any other combination of containers and values.

The root object of a property list is always an array or a dictionary.

class ViewController: UIViewController {
  override func viewDidLoad() {
    super.viewDidLoad()
    
    // create url to the file
    if let url = Bundle.main.url(forResource: "myPlist01", withExtension: "plist") {
      // create the decoder
      let decoder = PropertyListDecoder()
      do {
        // read the data
        let xml = try Data(contentsOf: url)
        // decode the data, note that the first parameter of the decode function
        // is [String : String].self to indicate we are decoding a [String : String]
        // dictionary
        let plistData = try decoder.decode([String : String].self, from: xml)
        // print the data for debugging purposes
        print(plistData)
      } catch {

      }

Types in Swift declare conformance to Encodable to support encoding, and Decodable to support decoding. If a type chooses to support both encoding and decoding, it can declare conformance to Codable.

Standard library types like String, Int, and Double; and Foundation types like Date, Data, and URL conform to Encodable and Decodable, which allows encoders and decoders such as PropertyListEncoder and PropertyListDecoder to encode and decode those data types.

When creating a custom type, encoding and decoding is supported if:

  1. The custom type declares conformance to Encodable and Decodable (or just Codable)
  2. The types of the properties in the custom type also conform to Encodable and Decodable

In the example below, Landmark meets the two requirements stated above:

  1. Landmark declares conformance to Codable
  2. The types of the properties in Landmark (String, Int and Coordinate) also conform to Codable.
struct Coordinate: Codable {
    var latitude: Double
    var longitude: Double
}

struct Landmark: Codable {
    // Double, String, and Int all conform to Codable.
    var name: String
    var foundingYear: Int
    
    // Adding a property of a custom Codable type maintains overall Codable conformance.
    var location: Coordinate
}

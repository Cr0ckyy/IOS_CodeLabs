NamedShape.java

public class NamedShape {
  int numberOfSides = 0;
  String name = "";

  NamedShape(String name) {
    this.name = name;
  }

  String simpleDescription() {
    return "A shape with " + numberOfSides + " sides.";
  }
}

Square.java

public class Square extends NamedShape {
  double sideLength = 0.0;

  Square(String name, double sideLength) {
    super(name);
    this.sideLength = sideLength;
    numberOfSides = 4;
  }

  double area() {
    return sideLength * sideLength;
  }

  String simpleDescription() {
    return "A square with sides of length " + sideLength + ".";
  }

  public double perimeter() {
    return sideLength * 4;
  }
}

public class MainApplication {

    System.out.println(square.area());

    System.out.println(square.numberOfSides);
  }
}

Classes are containers for data

Classes have properties, methods and initialisers:

  • properties are data associated with the object; properties should have a logical relationship to the class it belongs to
  • methods are behaviour associated with the object
  • initialisers are used to create an object

Defining a class

  • Define a class with the class keyword
  • Place the class definition within a pair of braces
class Square {
  // class definition goes here
}

Defining a property

  • Same syntax as declaring a variable
class Square {
  // class definition goes here

  // property
  var width = 0.0
}

Defining a method

  • Syntax will be covered in a later section
class Square {
  // class definition goes here

  // property
  var width = 0.0
  
  // method
  func area() -> Double {
    return width * width
  }
}

Creating an instance / object

  • Use the initializer syntax to create a new instance
  • The simplest form of initializer syntax uses the type name of the class followed by empty parentheses ()
// create an instance
var mySquare = Square()

Accessing properties

  • You can access the properties of an instance using dot syntax.
  • In dot syntax, you write the property name immediately after the instance name, separated by a period . without any spaces.
// accessing a property
print(mySquare.width)

Calling a method

  • You can call the method of an instance using dot syntax.
  • In dot syntax, you write the method name immediately after the instance name, separated by a period . without any spaces. The method name is followed by parentheses ()
// calling a method
print(mySquare.area())

https://docs.swift.org/swift-book/LanguageGuide/Properties.html

Properties associate values with a class

  • Stored properties store constant and variable values as part of an instance
  • Computed properties calculate (rather than store) a value

Stored properties

  • A stored property is a constant (introduced by the let keyword) or variable (introduced by the var keyword) that is stored as part of an instance of a class. 
class Square {
  // class definition goes here

  // stored properties
  var width = 0.0      // variable
  let color = "white"  // constant
}

Computed properties

  • Computed properties do not store a value; instead, they retrieve other properties indirectly
  • Declare the property with a type annotation followed by a pair of braces {}
  • Within the braces, add a return statement with the value to be returned
class Square {
  // class definition goes here

  // stored properties
  var width = 0.0      // variable
  let color = "white"  // constant

  // computed property
  var area: Double {
    return width * width
  }
}

https://docs.swift.org/swift-book/LanguageGuide/Functions.html

  • A function is a self-contained block of code that performs a specific task
  • The function is given a name that identifies what it does, and this name is used to "call" the function to perform its task when needed
  • A function may have 0 or more parameters and 0 or one return value

Anatomy of a function

func add(num1 x:Int, num2 y:Int) -> Int {
  // function body
  return x + y
}

// call the function
var sum = add(num1:3, num2:4)

If the function does not return any value, the section which indicates the return type is omitted

Each parameter has an argument label and a parameter name:

  • the argument label is used when calling the function; each argument is written in the function call with its argument label before it
  • the parameter name is used in the implementation of the function

Parameters with explicitly declared argument label and parameter name

// each parameter has an explicitly declared argument label and parameter name
func add(num1 x:Int, num2 y:Int) -> Int {
  // function body
  return x + y
}

// when calling the function, prefix the values with the argument labels
var sum = add(num1:3, num2:4)

Parameters with parameter name only

// each parameter has a parameter name only
// the parameter name is also used as the argument label
func add(x:Int, y:Int) -> Int {
  // function body
  return x + y
}

// when calling the function, prefix the values with the argument labels
var sum = add(x:3, y:4)

Parameters with argument label as _ and parameter name

// parameter has an _ for the argument label
func add(_ x:Int, y:Int) -> Int {
  // function body
  return x + y
}

// when calling the function, _ means the argument label is omitted
var sum = add(3, y:4)

Initialization prepares the object for use

  • Set an initial value for each stored property
  • Perform any other setup that is required

A default initializer with no parameters is provided for classes with no initializers

  • All properties must be initialized when declared

When one or more initializers are declared

  • No default initializer is provided
  • All properties must be initialized when declared or in the initializer
class Square {
  // class definition goes here

  // property
  var width : Double
  
  // method
  func area() -> Double {
    return width * width
  }
  
  // initializer
  init(width: Double) {
    // self.width refers to the property,
    // whereas width refers to the parameter
    self.width = width
  }
}

// create an instance
var mySquare = Square(width: 10)
// read the property value
print(mySquare.width)
// call the method
print(mySquare.area())

https://docs.swift.org/swift-book/LanguageGuide/Inheritance.html

  • Inheritance is when a class (the subclass) is based on another class (the parent class)
  • The subclass inherits the initializers, properties and methods of the parent class
  • Inheritence is known as the is a relationship; i.e. the subclass is a type of parent class (e.g. a square is a shape, a car is a vehicle, etc.)
  • To indicate that a class is a subclass of a superclass, add a colon and the superclass name after the class name
class Shape {
  var name : String

  init(name : String) {
    self.name = name
  }

  func printName() {
    print(name)
  }
}

// In this example:
// superclass is Shape, subclass is Square
// Square is based on Shape and inherits the properties,
// initializers and methods of Shape
class Square : Shape {
}

// the initializer for Square is inherited from Shape
var square = Square(name: "Square")
// the name property for Square is inherited from Shape
square.name = "square"
// the printName() method for Square is inherited from Shape
square.printName()

https://docs.swift.org/swift-book/LanguageGuide/Inheritance.html

A subclass can provide its own custom implementation of a method that it would otherwise inherit from a superclass. This is known as overriding.

The subclass must add the override keyword in front of the method being overridden

The subclass may choose to:

  • replace behaviour of the superclass method, or
  • add on behaviour to the superclass method; the subclass calls the superclass method by using the syntax super.methodName()- within a class, the keyword super refers to the superclass
class Shape {
  var name : String = ""
  
  func printName() {
    print("Shape name is \(name)")
  }
  
  init(name : String) {
    self.name = name
  }
}

class Circle : Shape {
  override func printName() {
    // replace behaviour
    print("Circle Shape")
  }
}

class Rectangle : Shape {
  override func printName() {
    // add on behaviour
    // call superclass method first
    super.printName()
    print("Rectangle Shape")
  }
}

var myCircle = Circle(name:"a circle")
myCircle.printName()

var myRectangle = Rectangle(name: "a rectangle")
myRectangle.printName()

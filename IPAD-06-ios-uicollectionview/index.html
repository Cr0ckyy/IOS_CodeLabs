UICollectionView

  • An object that manages an ordered collection of data items and presents them using customizable layouts.
  • Most common use for collection views is to present items in a grid-like arrangement
  • Each data item is represented by a UICollectionViewCell object

To present its content on-screen, a collection view cooperates with many different objects:

  • the data source object provides information about cell content
  • the delegate object handles user interaction
  • the layout object provides information (layout attributes) about cell layout
  • the collection view object merges the content and layout information to achieve the final appearance

  • check for the kind of supplementary view
  • for the header, dequeue the header cell, cast to the class of the header, configure the header and return the header
override func collectionView(_ collectionView: UICollectionView,
                             viewForSupplementaryElementOfKind kind: String,
                             at indexPath: IndexPath) -> UICollectionReusableView {
  if (kind == UICollectionView.elementKindSectionHeader) {
    // cast the header to the class of the header
    let header = collectionView.dequeueReusableSupplementaryView(ofKind: kind,
      withReuseIdentifier: "header", for: indexPath) as! HeaderView
    // configure the header
    // return the header
    return header
  } else if (kind == UICollectionView.elementKindSectionFooter) {
    // cast the footer to the class of the footer
    let footer = collectionView.dequeueReusableSupplementaryView(ofKind: kind,
      withReuseIdentifier: "footer", for: indexPath) as! FooterView
    // configure the footer
    // return the footer
    return footer
  }
  fatalError("\(kind) is not a recognized supplementary element")
}

The steps for creating a section footer are very similar. Check the Section Footer box in the attributes inspector and use similar code when the kind is UICollectionView.elementKindSectionFooter.

UICollectionViewFlowLayout

  • A layout object that organizes items into a grid with optional header and footer views for each section.
  • The items in the collection view flow from one row or column (depending on the scrolling direction) to the next, with each row comprising as many cells as will fit.
  • Cells can be the same sizes or different sizes.

Interitem spacing

  • The minimum interitem spacing is the minimum spacing to use between items in the same row.
  • The actual interitem spacing may be larger than that depending on the screen size

Line spacing

  • The minimum line spacing is the minimum spacing to use between lines of items in the grid.
  • The actual line spacing may be larger than that if the items are of different sizes

Section insets

  • Section insets are a way to adjust the space available for laying out cells
  • Insets can be used to insert space after a section's header view and before its footer view.
  • Insets can be used to insert space around the sides of the content.

Attributes Inspector

  • The cell size, header size, footer size, minimum cell (interitem) spacing, minimum line spacing and section insets can be set in the Size Inspector for the Collection View Flow Layout object
  • For the cell size to take effect, Estimate Size should be set to None

Attributes

var scrollDirection: UICollectionView.ScrollDirection

The scroll direction of the grid.

var minimumLineSpacing: CGFloat

The minimum spacing to use between lines of items in the grid.

var minimumInteritemSpacing: CGFloat

The minimum spacing to use between items in the same row.

var itemSize: CGSize

The default size to use for cells.

var sectionInset: UIEdgeInsets

The margins used to lay out content in a section

var headerReferenceSize: CGSize

The default sizes to use for section headers.

var footerReferenceSize: CGSize

The default sizes to use for section footers.

Vertical scrolling vs horizontal scrolling

The difference between interitem spacing and line spacing depends on the scroll direction:

  • In vertical scrolling, where items are laid out from left to right in a line (row) and continue to the line below, a is the interitem spacing and b is the line spacing.
  • In horizontal scrolling, where items are laid from top to bottom in a line (column) and continue to the line on the right, b is the interitem spacing and a is the line spacing

  • UICollectionView cell selection and highlighting

The UICollectionViewDelegate protocol contains methods which are called during the cell selection and cell highlighting process.

When the user taps a cell, the app has to determine the state of the cell and change the cell's appearance to indicate the transition between the 3 states: unselected, highlighted, and selected.

UICollectionView supports single and multiple selection. For multiple selection, set the allowsMultipleSelection property to true.

Single selection

For single selection, when the user clicks on a cell, the system will call collectionView(_:shouldSelectItemAt:) and check the return value.

If the return value is false, nothing happens.

If the return value is true:

  • If there is a previously selected cell, deselect the previously selected cell and call collectionView(_:didDeselectItemAt:) for the previously selected cell
  • Select the current cell and call collectionView(_:didSelectItemAt:)

Multiple selection

For multiple selection, when the user clicks on a cell, the system will perform the following sequence:

If the cell is currently deselected, iOS will call collectionView(_:shouldSelectItemAt:) and check the return value.

  • If the return value is false, nothing happens.
  • If the return value is true, the cell will be selected and iOS will call collectionView(_:didSelectItemAt:)

If the cell is currently selected, iOS will call collectionView(_:shouldDeselectItemAt:) and check the return value.

  • If the return value is false, nothing happens.
  • If the return value is true, the cell will be deselected and iOS will call collectionView(_:didDeselectItemAt:)

Highlighting

A cell is highlighted when the user presses the cell and does not release. The highlighted cell becomes unhighlighted when the press is released.

When the cell is pressed, iOS will call collectionView(_:shouldHighlightItemAt:).

  • If the return value is false, nothing happens.
  • If the return value is true, the cell will be highlighted and iOS will call collectionView(_:didHighlightItemAt:)

When the press is released, iOS will call collectionView(_:didUnhighlightItemAt:)

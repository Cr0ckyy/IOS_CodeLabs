The Xcode workspace is divided into the following areas

  • Navigator - provides quick access to different parts of the project
  • Editor - edit the object selected in the navigator
  • Inspector - view and edit information for an object selected in the navigator or editor area
  • Toolbar - used to build and run the app, view the progress of running tasks, and configure the work environment
  • Debug - used to control the execution of your app and inspect variables

The navigator bar at the top of the navigator area has icons for the different navigators, while the inspector bar at the top of the inspector area has icons for the different inspectors. Mouse over the icons to discover the various navigators and inspectors.

IBOutlet properties in the view controller are used to access user interface objects in Main.storyboard.

  1. Select Main.storyboard
  2. Show the Assistant Editor
  3. Right-click drag from the user interface object in Main.storyboard (via the Document Outline) to ViewController.swift
  4. In the Connection dialog, provide a name for the IBOutlet property and click Connect

import UIKit
class ViewController: UIViewController {
  // Put IBOutlet properties at the start of the class
  @IBOutlet weak var outLabel: UILabel!

  override func viewDidLoad() {
    super.viewDidLoad()
  }
}

Notes

  • Starting all IBOutlet property names with out helps narrow the autocomplete suggestions when we write our code
  • Provide a name for the IBOutlet property which can be easily understood by anyone reading your code about which user interface object is connected to that property
  • Right-click drag from the user interface object via the Document Outline; it is quite easy to select the wrong item from the canvas
  • Double-check the type from the Connection dialog; if it shows a different type from what you expected, the wrong item may have been selected
  • Put IBOutlet properties at the start of the class
  • Create IBOutlet properties for user interface objects which will have their attributes read or updated when the application is executed

https://developer.apple.com/documentation/uikit/uislider

A control used to select a single value from a continuous range of values.

Responding to User Interaction

  • Sliders use the Target-Action design pattern to notify your app when the user moves the slider. To be notified when the slider's value changes, register your action method with the valueChanged event. At runtime, the slider calls your method in response to the user changing the slider's value.

Properties / Attributes

  • value: Float - slider's current value
  • minimumValue: Float - minimum value of the slider
  • maximumValue: Float - maximum value of the slider

IBAction methods in the view controller are used to respond to user events experienced by user interface objects in Main.storyboard.

  1. Select Main.storyboard
  2. Show the Assistant Editor
  3. Right-click drag from the user interface object in Main.storyboard to ViewController.swift
  4. In the Connection dialog, select the type, the event to respond to and provide a name for the IBAction method and click Connect

import UIKit
class ViewController: UIViewController {
  override func viewDidLoad() {
    super.viewDidLoad()
  }
  
  // Put IBAction methods at the end of the class
  @IBAction func actSlider(_ sender: UISlider) {
  }  
}

Notes

  • Provide a name for the IBAction method which can be easily understood by anyone reading your code about which user interface object is connected to that method and what event causes that method to be executed
  • Only user interface objects which can respond to user events can be connected to an IBAction method
  • Double-check the type from the Connection dialog; if it shows a different type from what you expected, the wrong item may have been selected
  • The parameter in the IBAction method is the object that experienced the event
  • Put IBAction methods at the end of the class

Connections inspector is used to view and delete the IBOutlet properties and IBAction methods associated with a user interface object in Main.storyboard.

Caution: To remove an IBOutlet property or IBAction method, delete the code from ViewController.swift and break the connection by clicking on the x next to the IBOutlet property or IBAction method in the connections inspector. If there is an inconsistency between the information in Main.storyboard and ViewController.swift, the application is likely to crash.

Connections between user interface objects in Main.storyboard and IBOutlet properties in ViewController.swift appear under Referencing Outlets

Connections between user interface objects in Main.storyboard and IBAction methods in ViewController.swift appear under Sent Events

import UIKit
// ViewController is a subclass of UIViewController which is the default view controller class
class ViewController: UIViewController {
  
  // Put IBOutlet properties at the start of the class
  @IBOutlet weak var outLabel: UILabel!
  
  // Overridden method viewDidLoad() is called when the view is first loaded
  override func viewDidLoad() {
    super.viewDidLoad()
    // set the text for the label
    outLabel.text = "Hello"
  }
  
  // Put IBAction methods at the end of the class
  @IBAction func actSlider(_ sender: UISlider) {
    // set the text for the label with the value of the slider
    outLabel.text = "\(sender.value)"
  }
}
  1. Run the project. Move the slider and observe the text in the UILabel object.
